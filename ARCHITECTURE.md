# ARCHITECTURE.md

## Текущее состояние

SkillCheck сейчас работает как статический сайт на GitHub Pages с подключением к Google Apps Script.

Работают 2 теста из 5:

1. Financial Analyst Junior.
2. Credit Analyst Junior.

Ещё 3 теста присутствуют в интерфейсе и структуре файлов, но требуют полноценной доработки банков вопросов:

1. FP&A / Budget Analyst Junior.
2. Accounting Junior.
3. Finance BI / Data Analyst Junior.

## Общая схема

```text
Пользователь
   ↓
index.html
   ↓ выбор теста
test.html?test=<testId>
   ↓ загрузка вопроса
/data/<testId>.json
   ↓ завершение теста
Google Apps Script
   ↓
Google Sheets + Google Drive TXT report
```

## Роли файлов

### `index.html`

Главная страница сайта. Отвечает за:

- дизайн лендинга;
- карточки тестов;
- переход на нужный тест;
- позиционирование проекта.

### `test.html`

Главный движок прохождения теста. Отвечает за:

- чтение параметра `?test=` из URL;
- выбор нужной конфигурации теста;
- загрузку JSON-файла с вопросами;
- регистрацию кандидата;
- показ вопросов по одному;
- таймер на каждый вопрос;
- перемешивание вопросов и ответов;
- подсчёт результата;
- фиксацию уходов со вкладки;
- запрет копирования;
- генерацию TXT-отчёта;
- отправку результата в Google Apps Script.

### `admin.html`

Черновая админ-панель. Сейчас её не нужно считать основной системой управления результатами. Основная рабочая таблица — Google Sheets.

В будущем `admin.html` можно превратить в панель работодателя.

### `data/*.json`

Банки вопросов. Каждый файл соответствует отдельному тесту.

Текущие файлы:

```text
data/fa-junior.json   Financial Analyst Junior
data/ca-junior.json   Credit Analyst Junior
data/fpa-junior.json  FP&A Junior
data/acc-junior.json  Accounting Junior
data/bi-junior.json   Finance BI / Data Analyst Junior
```

## Логика выбора теста

В `test.html` есть объект `TEST_CONFIG`. Он связывает `testId` с названием, описанием, версией, файлом вопросов и блоками навыков.

Пример:

```javascript
"ca-junior": {
  title: "Credit Analyst Junior",
  description: "Ликвидность, долговая нагрузка, cash flow, кредитный риск",
  version: "CA Junior v1.0",
  questionsFile: "data/ca-junior.json",
  blocks: {...}
}
```

## Текущая логика прохождения

1. `DOMContentLoaded` читает параметр `test` из URL.
2. Если тест найден в `TEST_CONFIG`, загружается соответствующий JSON.
3. Если JSON содержит `questionsPerTest`, сайт показывает только часть вопросов.
4. Перед стартом кандидат вводит имя, email, телефон и уровень английского.
5. После старта показывается один вопрос.
6. После ответа переход назад невозможен.
7. Если время вопроса вышло, ответ фиксируется как `Нет ответа` или `Время вышло`.
8. После последнего вопроса считается результат.

## Античит

Сейчас реализовано:

- фиксация ухода со вкладки через `visibilitychange`;
- фиксация потери фокуса через `blur`;
- защита от двойного засчитывания через `lastTabExitAt`;
- запрет контекстного меню;
- запрет копирования;
- запрет `Ctrl+C`, `Ctrl+U`, `Ctrl+Shift+I`, `Ctrl+Shift+J`, `F12`;
- один вопрос на экран;
- ограничение времени на вопрос.

Ограничение: веб-сайт не может полностью запретить скриншот или съёмку экрана на телефон.

## Скоринг

Сайт считает:

```text
rawScore = сумма набранных баллов
rawTotal = максимальная сумма баллов
percent = rawScore / rawTotal * 100
penalty = штраф за уходы со вкладки
finalScore = percent - penalty
```

Плашки:

```text
Junior Strong      finalScore >= 85 и мало уходов
Junior Confirmed   finalScore >= 70 и умеренное поведение
Borderline         finalScore >= 60
Not Confirmed      finalScore < 60
```

Также считается `Trust Score`, который учитывает итоговый балл, уходы и количество неотвеченных вопросов.

## Google Apps Script

Apps Script должен делать 4 вещи:

1. Принимать POST-запрос с результатом.
2. Создавать TXT-отчёт в Google Drive.
3. Писать строку результата в Google Sheets.
4. Проверять повторные попытки, если `RETAKE_CHECK_ENABLED = true`.

Обновлённый код лежит в:

```text
apps-script/Code.gs
```

## Нужная структура Google Sheets

Колонки должны быть такими:

```text
Дата
Тест
Версия
Имя
Email
Телефон
Английский
Уход со вкладки
Баллы
Всего
Процент
Итоговый балл
Плашка
Статус
Trust Score
Ссылка на TXT отчет
```

Нужно убрать старые колонки:

```text
Решение
Баллы (сырые)
Всего (сырые)
```

Почему: они перегружают таблицу. Подробности должны оставаться в TXT-отчёте.

## TXT-отчёт

TXT-отчёт создаётся на стороне клиента в `test.html`, затем отправляется в Apps Script и сохраняется в Google Drive.

Отчёт включает:

- профиль теста;
- данные кандидата;
- итоговый результат;
- skill card по блокам;
- детальный разбор вопросов;
- уходы со вкладки;
- рекомендацию.
