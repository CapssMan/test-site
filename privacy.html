Политика обработки персональных данных

Эта политика описывает, какие данные обрабатывает SkillCheck / finance-assessment-platform в рамках MVP-тестирования кандидатов.

Результат тестирования является предварительной оценкой отдельных навыков кандидата и не является самостоятельным решением о найме, отказе в найме или профессиональной пригодности. Итоговое решение всегда должно приниматься человеком с учётом собеседования, опыта, портфолио и иных факторов.

1. Кто обрабатывает данные

Оператор сайта SkillCheck / finance-assessment-platform.

Реквизиты оператора будут уточнены при коммерческом запуске проекта.

2. Какие данные обрабатываются

  • имя;
  • email;
  • Telegram — добровольно предоставляемое и необязательное контактное поле;
  • уровень английского;
  • опыт;
  • источник кандидата;
  • выбранный тест;
  • ответы на вопросы;
  • результат теста;
  • итоговый балл;
  • уходы со вкладки;
  • дата и время прохождения;
  • технический идентификатор браузера;
  • технические данные прохождения, необходимые для защиты от повторных попыток.

3. Цели обработки

  • проведение тестирования;
  • формирование результата;
  • создание TXT-отчёта;
  • предотвращение повторных попыток;
  • защита от недобросовестного прохождения;
  • анализ качества тестов;
  • связь с кандидатом по email, если это потребуется по результатам тестирования;
  • быстрая обратная связь через Telegram, только если кандидат добровольно указал его.

4. Технический идентификатор браузера

Для защиты от повторного прохождения и недобросовестного использования сайт формирует технический идентификатор браузера. Сырые параметры браузера не публикуются и не передаются третьим лицам как отдельный отчёт. В результатах хранится только hash технического идентификатора.

5. Где хранятся данные

  • закрытые папки Яндекс Диска;
  • локальное хранилище браузера — только для временной резервной копии неподтверждённого результата;
  • GitHub Pages как сайт;
  • Google Apps Script как backend.

Telegram указывается добровольно, не требуется для прохождения теста и используется только для связи с кандидатом. Если результат успешный, нормализованный Telegram сохраняется только в полном TXT-отчёте. Он не включается в обезличенную аналитику, записи контроля повторных попыток и публичную админ-панель.

Если отправка результата временно не подтверждена сервером, полная резервная копия результата сохраняется только в браузере кандидата не более чем на 24 часа. Она удаляется после подтверждённой отправки или по истечении этого срока.

6. Трансграничная передача

Отдельные сервисы, используемые для работы сайта и хранения результатов, могут относиться к зарубежной инфраструктуре. При развитии проекта оператор должен дополнительно оценить необходимость уведомления уполномоченного органа о трансграничной передаче персональных данных.

7. Срок хранения

Данные хранятся до достижения целей обработки, но не более 12 месяцев, если более длительный срок не требуется для защиты прав оператора или анализа качества тестирования.

8. Передача результатов работодателям

Результаты тестирования не передаются работодателям, партнёрам проекта или третьим лицам без отдельного согласия кандидата.

9. Меры защиты данных

  • доступ к таблицам и отчётам ограничивается;
  • код результата не является ключом доступа к полному TXT-отчёту; полный отчёт доступен только после отдельной административной авторизации;
  • данные не публикуются в открытом доступе;
  • не собираются паспортные данные, адрес, дата рождения и иные избыточные сведения;
  • технический идентификатор браузера хранится в виде hash;
  • данные хранятся ограниченный срок.

10. Важные ограничения MVP

Проект находится на стадии MVP. При коммерческом запуске оператору необходимо дополнительно оценить требования законодательства о персональных данных, включая уведомление уполномоченного органа, вопросы локализации баз данных и трансграничной передачи персональных данных.

11. Права пользователя

Пользователь может запросить:

  • уточнение данных;
  • удаление данных;
  • отзыв согласия;
  • информацию об обработке.
