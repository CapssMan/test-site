# SkillCheck

SkillCheck — MVP-платформа для первичного отбора junior-кандидатов в финансовых направлениях.

Проект проверяет не «красивое резюме», а практические навыки: понимание финансовой логики, работу с отчётностью, базовую аналитику, внимательность к рискам и способность быстро принимать решения по кейсам.

## Текущий статус

Рабочая версия сайта уже включает выбор тестов, прохождение теста по одному вопросу, таймеры, античит-механику, отправку результатов в Google Sheets и генерацию TXT-отчёта в Google Drive.

Сейчас в работе 5 направлений:

| Тест | Статус | Файл банка вопросов |
|---|---|---|
| Финансовый аналитик Junior | Работает | `data/fa-junior.json` |
| Кредитный аналитик Junior | Работает / дорабатывается | `data/ca-junior.json` |
| FP&A / Budget Analyst Junior | Черновик | `data/fpa-junior.json` |
| Accounting Junior | Черновик | `data/acc-junior.json` |
| Finance BI / Data Analyst Junior | Черновик | `data/bi-junior.json` |

## Цель MVP

Сделать 5 тестов для массового первичного отбора junior-специалистов в финансах, затем начать привлекать кандидатов и показывать работодателям рабочий MVP.

Главная ценность продукта:

- быстро отсеивать слабых кандидатов;
- находить сильных junior-кандидатов;
- давать работодателю короткий skill-report по кандидату;
- снижать число бесполезных собеседований;
- проверять практическую финансовую логику, а не только определения.

## Как работает сайт

1. Пользователь открывает `index.html`.
2. Выбирает нужный тест.
3. Переходит на `test.html?test=<testId>`.
4. Заполняет имя, email, телефон и уровень английского.
5. Проходит тест по одному вопросу.
6. Сайт фиксирует ответы, время и уходы со вкладки.
7. Результат отправляется в Google Apps Script.
8. Apps Script записывает данные в Google Sheets.
9. Apps Script создаёт TXT-отчёт в Google Drive и пишет ссылку в таблицу.

## Запуск локально

Так как вопросы загружаются через `fetch()` из JSON-файлов, лучше запускать проект через локальный сервер, а не просто открывать HTML-файл двойным кликом.

Простой вариант:

```bash
python -m http.server 8000
```

После этого открыть:

```text
http://localhost:8000
```

## Деплой

Проект можно держать на GitHub Pages.

Минимальный flow:

1. Изменить файлы.
2. Закоммитить изменения.
3. Запушить в GitHub.
4. Открыть сайт через GitHub Pages.
5. Обновить страницу через `Ctrl + F5`.

## Важные правила разработки

1. Не хранить все вопросы внутри `index.html` или `test.html`.
2. Каждый тест держать отдельным JSON-файлом в папке `data/`.
3. Не делать правильный ответ самым длинным или самым очевидным.
4. У каждого вопроса должно быть 4 варианта ответа.
5. Варианты должны быть примерно одинаковой длины.
6. Неправильные варианты должны быть правдоподобными.
7. Сложные объяснения хранить в `comment`, а не в самом варианте ответа.
8. Перед изменением Apps Script делать новую версию deployment.
